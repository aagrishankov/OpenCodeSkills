# Примеры для skill-description-generator

## 1) Happy Path

Вход:

```text
Обнови README по всем SKILL.md, единый формат, перевод на русский.
```

Ожидаемое поведение:

- Скилл находит все `SKILL.md`.
- Берет `metadata.short-description` как базу для "Назначение".
- Обновляет раздел `## Skills` по единому шаблону.

## 2) Плохой пример

Плохой результат:

- Записи в README в разном формате.
- Часть навыков без блока "Назначение".
- Описания оставлены на английском без причины.

Почему это ошибка:

- Нарушен единый шаблон.
- Нарушен metadata-first и ru-localization подход.

## 3) Исправленная версия

- Все записи приведены к одной структуре.
- "Назначение" заполнено на русском.
- Для отсутствующих метаданных применен fallback.

## 4) Edge Case

Сценарий:

```text
У нескольких SKILL.md нет metadata.short-description.
```

Ожидаемое поведение:

- Использовать первую фразу `description`.
- Если и ее нет, взять краткую формулировку из `## 1) Purpose`.

## 5) Граница триггера

Запрос:

```text
Сгенерируй Kotlin usecase слой.
```

Ожидаемое поведение:

- Этот скилл не срабатывает, потому что запрос не про `README.md` и не про каталог описаний скиллов.
