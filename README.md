# OpenCodeSkills

## Skills

- `skill-creator` - создание и улучшение скиллов для OpenCode: помогает собрать требования, написать `SKILL.md`, настроить структуру ресурсов (`scripts/`, `references/`, `assets/`), улучшить описание для корректного триггера и итеративно доработать скилл по обратной связи.
- `kotlin-response-request` - генерация Kotlin `Request`/`Response` DTO на `kotlinx.serialization`: все поля всегда nullable (опциональные) и со значением по умолчанию `null`, обязательны аннотации `@Serializable` для классов и `@SerialName` для полей, используются только `data class`, допускается только вложенность `root -> nested`, вложенность `nested -> nested` запрещена, и в одном `.kt` файле допускается только одна root-модель.
- `kotlin-safe-model-mapper` - генерация безопасных Kotlin-моделей приложения и детерминированных мапперов из небезопасных transport-моделей: строго `toData()`/`toRequest()`, один root-маппер на файл, `internal` для root и `private` для helper-функций, только block body, обязательные trailing comma, нормализация строк, фильтрация невалидных nested-элементов, агрегация с сохранением исходных полей, архитектурная нейтральность и встроенные шаблоны/примеры/чеклисты.
- `kotlin-client-repository-generator` - генерация compile-ready repository-слоя для клиентских Kotlin приложений (Android/KMP): интерфейс и реализация репозитория, корректные зависимости (`featureDao`, `httpClient`) только при использовании, вызовы существующих mapper-ов (`toData()`/`toRequest()`/`toEntity()`), обязательные KDoc (краткий в интерфейсе и расширенный в реализации), импорты, шаблоны для DAO-only / HTTP-only / DAO+HTTP / Flow / HTTP 204 и строгие ограничения без DI/threading/error-wrapping/paging.
- `kotlin-client-usecase-generator` - генерация клиентских Kotlin use case для Android/KMP: интерфейс и `internal`-реализация `UseCase/UseCaseImpl`, строгий контракт `suspend execute(...)`, обязательный `DispatcherProvider` + `withContext` (IO по умолчанию), orchestration-логика с optional private helper methods, KDoc на русском, named arguments, trailing comma, шаблоны и чеклист без генерации repository/mapper/DI/Flow.
