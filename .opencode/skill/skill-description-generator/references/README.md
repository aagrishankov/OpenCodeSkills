# Skill Description Generator References

Этот раздел описывает, как обновлять `README.md` безопасно и единообразно.

## Границы обновления

- Обновляется только раздел `## Skills`, если пользователь явно не просит иное.
- Сначала используется frontmatter (`name`, `description`, `metadata.*`).
- При отсутствии `metadata.short-description` применяется fallback из `description`.

## Источники данных (приоритет)

1. `metadata.short-description`
2. Первая фраза `description`
3. Первая осмысленная строка из `## 1) Purpose`

## Правило локализации

- Текст назначения переводится на русский.
- Технические токены и имена (`DTO`, `Request`, `Response`, `SKILL.md`) сохраняются.

## Единый формат

- Для каждой записи используется шаблон из `assets/readme-entry-template.md`.
- Формат должен быть одинаковым для всех навыков в рамках одного обновления.
